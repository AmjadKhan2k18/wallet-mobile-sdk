Encryption

MWP uses deep links for direct communication between peers.

To ensure security, all messages in MWP after secure key exchange process are end-to-end encrypted.

Key generationâ€‹

Both the client and the wallet generate their own key pair for each session via Key-agreement protocol using Elliptic-curve.

Each peer stores its private key in secure persistent storage and share its public key with the other party.

Key exchange to derive shared secretâ€‹

MWP uses Diffieâ€“Hellman key exchange method to securely share cryptographic keys over deep links.

This method allows the two parties to derive the same shared secret offline using its own private key and the peer's public key.

Message content encryptionâ€‹

After successful handshake process to exchange keys, subsequent messages (content data) are encrypted with the common shared secret using a symmetric-key algorithm.

It uses AES-GCM algorithm for cryptographic operations.

Messages without encryptionâ€‹

However, there are two exceptions where the content data are not encrypted:

  1. Handshake calls from the client in order to exchange keys with the wallet
  2. Failure responses from wallet to return errors happening during the handshake processes

{
  "version": "1.0.3",

  "content": {
    "handshake": {
      "appId": "com.coinbase.SampleClient",
      "callback": "myappxyz://mycallback/wsegue",
      "initialActions": [
        {
          "paramsJson": "{}",
